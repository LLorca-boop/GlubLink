# GlubLink - Руководство по Запуску

## 📋 Предварительные Требования

### Системные требования
- **ОС**: Windows 10/11 (для Hard Links) или macOS/Linux (без Hard Links)
- **Flutter**: 3.16.0+
- **Dart**: 3.2.0+
- **Память**: Минимум 4 GB RAM (рекомендуется 8 GB)
- **Место на диске**: 500 MB для приложения + место для кэша

### Для работы с Hard Links (Physical Mode)
- **Файловая система**: NTFS (обязательно для Windows)
- **Режим разработчика**: Включён в Windows 10/11
- **Права доступа**: Чтение/запись в выбранную папку

## 🔧 Режимы Организации Файлов

### Virtual Mode (по умолчанию)
- Файлы остаются на своих местах
- Категоризация через базу данных
- Максимальная скорость и безопасность
- **Рекомендуется для большинства пользователей**

### Physical Mode (Hard Links)
- Создаются Hard Links в папках тегов
- Файлы физически организуются по категориям
- Требует NTFS и прав разработчика
- **Внимание**: Не работает между разными дисками

#### Переключение режима
1. Откройте настройки
2. Выберите **"File Organization Mode"**
3. Выберите нужный режим
4. Подтвердите миграцию

## 🔍 Поиск и Фильтрация

### Операторы поиска
- **Обычный поиск**: `tag1 tag2` - найти медиа со всеми тегами
- **Исключение**: `-tag1` - исключить медиа с этим тегом
- **Логическое ИЛИ**: `(tag1||tag2||tag3)` - найти медиа с любым из тегов

### Примеры
```
cat dog -bird        # Кошки и собаки, без птиц
(red||blue||green)   # Красное, синее или зелёное
artist:john -nsfw    # Работы John без NSFW
```

## 🎨 Режимы Отображения

### Justified Layout
- Горизонтальная сетка

- **Лучше для**: Pinterest-style галереи

## 🛠️ Решение Проблем

### Ошибка: "Hard Links not supported"
**Причина**: FAT32/exFAT вместо NTFS  
**Решение**: 
1. Проверьте файловую систему: `fsutil fsinfo volumeinfo C:`
2. Преобразуйте в NTFS или используйте Virtual Mode

### Ошибка: "Developer Mode required"
**Причина**: Режим разработчика Windows выключен  
**Решение**:
1. Откройте Параметры Windows
2. Обновление и безопасность → Для разработчиков
3. Включите "Режим разработчика"

### Ошибка: "API Key not set"
**Причина**: Не настроены API ключи  
**Решение**: Настройте ключи в настройках приложения

### Ошибка: "Database corrupted"
**Причина**: Повреждение БД  
**Решение**:
1. Приложение автоматически восстановит из бэкапа
2. Если не помогло: удалите `%APPDATA%/GlubLink/database`
3. Перезапустите приложение

### Файлы не отображаются
**Причина**: Файлы удалены вне приложения  
**Решение**:
1. Откройте галерею
2. Нажмите "Refresh"
3. Удалите битые ссылки через настройки

## 🔐 Безопасность

### API Ключи
- Хранятся в Windows Credential Locker
- Шифруются через DPAPI
- Маскируются в логах (***)

### База Данных
- ACL (Access Control List)
- Доступ только пользователю-владельцу
- Автоматическое резервное копирование каждые 4 часа

## 📦 Резервное Копирование

### Автоматический бэкап
- **Частота**: Каждые 4 часа
- **Хранение**: 7 последних версий локально
- **Расположение**: `%APPDATA%/GlubLink/backups/`

### Ручной экспорт
1. Настройки → Backup & Restore
2. Нажмите "Export Data"
3. Выберите формат (JSON + медиа опционально)
4. Сохраните файл

### Восстановление
1. Настройки → Backup & Restore
2. Нажмите "Import Data"
3. Выберите файл бэкапа
4. Подтвердите восстановление
